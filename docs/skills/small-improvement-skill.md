# 小さい改善案スキル

いきなりファイルを変更せず、小さい改善案だけを出すための手順書です。

## 使う場面

どこを直せばよいかわからないとき、まず安全な改善案を見たいときに使います。

## Codexへの依頼例

このファイルについて、小さい改善案を3つまで出してください。
各案に、変更ファイル・理由・確認方法を付けてください。
最後に、初心者におすすめの1案を選んでください。
まだファイルは変更しないでください。

## 出してほしい内容

改善案は3つまでにします。

各案には、次の3つを入れます。

1. 変更ファイル
2. 理由
3. 確認方法

最後に、初心者におすすめの1案を選びます。

## 注意

- 大きな設計変更は避ける
- 複数ファイルを一気に変えない
- まずは見た目や文章など、確認しやすい改善を優先する
- 実装する前に、どの案を選ぶか確認する
